# 8.3 Avaliação de Aceitação e Qualidade

## Avaliação segundo ISO/IEC 25010

A norma ISO/IEC 25010 (SQuaRE) define um modelo de qualidade de software com 8 características principais. O SGCLC v1.0 foi avaliado segundo as 7 características mais relevantes para o contexto do projecto:

| Característica | Sub-características avaliadas | Avaliação (1–5) | Justificação |
|---|---|---|---|
| **Adequação Funcional** | Completude, Correcção | **4/5** | 38/42 RFs implementados (90,5%); todos os fluxos críticos funcionam sem defeitos bloqueantes |
| **Eficiência de Desempenho** | Comportamento temporal, Utilização de recursos | **3/5** | Respostas < 2s em dev (SQLite); desempenho em produção com SQL Server seria melhor |
| **Compatibilidade** | Interoperabilidade | **4/5** | API web padrão (HTTPS); interoperabilidade com SQL Server via EF Core |
| **Usabilidade** | Aprendibilidade, Operabilidade | **5/5** | POS operável em < 60s por utilizador não treinado (validado); interface intuitiva com Bootstrap |
| **Fiabilidade** | Disponibilidade, Tolerância a falhas | **4/5** | Retry automático de consolidação; transacções ACID; sem falha durante testes |
| **Segurança** | Confidencialidade, Autenticidade | **4/5** | BCrypt cost=12; RBAC completo; passwords nunca em texto claro |
| **Manutenibilidade** | Modularidade, Testabilidade, Modificabilidade | **4/5** | Arquitectura 3 camadas; interfaces; 10 testes unitários; baixo acoplamento |

**Avaliação global: 4,0/5,0** — o SGCLC v1.0 apresenta qualidade alta nas dimensões de Usabilidade, Segurança e Manutenibilidade, com espaço de melhoria em Desempenho (optimização de queries) e Fiabilidade (maior cobertura de testes de integração).

## Testes de Aceitação com o Cliente

Na sessão de demonstração realizada após a implementação, o cliente João Martins avaliou os seguintes cenários:

| Cenário | Critério de Aceitação | Resultado |
|---|---|---|
| Demonstração do Dashboard Central | "Quero ver o total de vendas de todas as lojas hoje" | ✅ Aceite — KPIs visíveis na página principal |
| Demonstração dos Alertas de Stock | "Quero ser alertado quando um produto estiver perto do mínimo" | ✅ Aceite — alertas visíveis no Dashboard e na página de Stock |
| Demonstração do POS | "O registo de venda tem de ser rápido" | ✅ Aceite — venda de 3 produtos em < 45 segundos |
| Demonstração da Consolidação | "A consolidação tem de ser automática" | ✅ Aceite — job activo e resultado visível em Consolidações |
| Pesquisa de produto sem acento | "cafe" deve encontrar "Café Expresso" | ✅ Aceite — funcionalidade verificada ao vivo |

**Resultado da demonstração:** O cliente aprovou o sistema e validou que todos os objectivos de negócio identificados no diagnóstico inicial (P1–P8) foram resolvidos no SGCLC v1.0, à excepção de RF30 (exportação de faturas para PDF) que ficou pendente para v1.1.
