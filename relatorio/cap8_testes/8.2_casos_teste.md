## Exemplo de Teste — ST-02 (Stock Negativo)

```csharp
[Fact]
public async Task AjustarStock_QuantidadeNegativa_DeveLancarExcecao()
{
    // Arrange
    var stock = new Stock { Id = 1, LojaId = 1, ProdutoId = 1,
                            Quantidade = 100, StockMinimo = 20 };
    _mockRepo.Setup(r => r.GetByLojaAndProdutoAsync(1, 1))
             .ReturnsAsync(stock);

    // Act & Assert
    await Assert.ThrowsAsync<InvalidOperationException>(
        () => _service.AjustarStockAsync(1, 1, -200, "Motivo", "user1")
    );
}
```

## Testes Manuais — Fluxos Principais

| Fluxo | Pré-condições | Passos | Resultado Esperado | Resultado Obtido |
|---|---|---|---|---|
| Login válido | Utilizador "carlos@qm.pt" activo | Inserir email/password → Entrar | Redirect para Dashboard POS | ✅ |
| Login inválido | — | Inserir password errada | Mensagem "Credenciais inválidas" | ✅ |
| Venda POS completa | Produto com stock > 0 | Filtrar produto → Adicionar → Confirmar | Venda registada + stock debitado + recibo | ✅ |
| Venda com stock insuficiente | Produto com stock = 1 | Tentar vender 10 unidades | Mensagem de erro, venda não processada | ✅ |
| Ajuste de stock negativo (bloqueio) | Stock = 10 | Tentar ajuste de -50 | Erro "Stock não pode ficar negativo" | ✅ |
| Nova encomenda + receção | Fornecedor activo | Criar encomenda → Receber → Verificar stock | Stock aumentado com qtd recebida | ✅ |
| Consolidação manual | Utilizador GestorCadeia | Clicar "Consolidar Agora" | Registo em Consolidacoes com estado Sucesso | ✅ |
| Pesquisa accent-insensitive | Produto "Café" no sistema | Pesquisar "cafe" | "Café Expresso" aparece nos resultados | ✅ |
| RBAC — Funcionário sem acesso a Stock | Login como Funcionário | Tentar navegar para /stock | Redirect para login ou página de acesso negado | ✅ |
