# 7.2 Implementação da Base de Dados

## AppDbContext

O `AppDbContext` em `ConvenienceChain.Data/AppDbContext.cs` é o ponto central de acesso à base de dados. Define os *DbSets* para cada entidade e configura as relações e restrições via Fluent API:

```csharp
public class AppDbContext : DbContext
{
    public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

    public DbSet<Loja> Lojas { get; set; }
    public DbSet<Utilizador> Utilizadores { get; set; }
    public DbSet<Produto> Produtos { get; set; }
    public DbSet<Categoria> Categorias { get; set; }
    public DbSet<Stock> Stocks { get; set; }
    public DbSet<Venda> Vendas { get; set; }
    public DbSet<LinhaVenda> LinhasVenda { get; set; }
    public DbSet<Encomenda> Encomendas { get; set; }
    public DbSet<LinhaEncomenda> LinhasEncomenda { get; set; }
    public DbSet<Fatura> Faturas { get; set; }
    public DbSet<Fornecedor> Fornecedores { get; set; }
    public DbSet<Consolidacao> Consolidacoes { get; set; }
    public DbSet<AjusteStock> AjustesStock { get; set; }

    protected override void OnModelCreating(ModelBuilder mb)
    {
        // Restrição de unicidade: um produto por loja
        mb.Entity<Stock>()
            .HasIndex(s => new { s.LojaId, s.ProdutoId })
            .IsUnique();

        // Restrição de unicidade: uma consolidação por loja por dia
        mb.Entity<Consolidacao>()
            .HasIndex(c => new { c.LojaId, c.DataConsolidacao })
            .IsUnique();
    }
}
```

## SeedData

O `SeedData.Initialize()` preenche a base de dados com dados iniciais representativos para a QuickMart. É chamado no arranque da aplicação em `Program.cs`:

```csharp
// Em Program.cs, após Build():
using (var scope = app.Services.CreateScope()) {
    var context = scope.ServiceProvider.GetRequiredService<AppDbContext>();
    context.Database.EnsureCreated();
    SeedData.Initialize(context);
}
```

O SeedData cria:
- 5 lojas QuickMart (Braga-Centro, Palmeira, Maximinos, Barcelos, Guimarães)
- 3 utilizadores com papéis distintos (password hasheada com BCrypt custo 12)
- 5 categorias com hierarquia (Bebidas → Refrigerantes, Bebidas → Cafetaria...)
- 20 produtos com preços e códigos únicos
- Stock inicial para cada produto em cada loja com StockMinimo definido
- 2 fornecedores activos

## Migrações e Esquema Físico

Na versão de desenvolvimento com SQLite, a base de dados é criada automaticamente via `EnsureCreated()`. Para ambientes de produção com SQL Server, as migrações EF Core são geradas com:

```bash
dotnet ef migrations add InitialCreate --project src/ConvenienceChain.Data/ \
    --startup-project src/ConvenienceChain.Web/
dotnet ef database update
```

O script DDL SQL completo está documentado em `docs/etapa2/dicionario_dados.md §SQL DDL` e pode ser utilizado directamente em SQL Server 2019+.
