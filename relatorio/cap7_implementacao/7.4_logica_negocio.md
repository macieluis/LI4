# 7.4 Camada de Lógica de Negócio

## Serviços de Aplicação

A camada de lógica de negócio do SGCLC é implementada em classes de serviço (`*Service`) registadas como *scoped* no contentor de DI do ASP.NET Core. Cada serviço encapsula as regras de negócio de um módulo funcional e coordena os repositórios necessários.

## Serviço de Autenticação — `AuthService`

O `AuthService` é responsável pela autenticação dos utilizadores e pela criação de sessão:

```csharp
public async Task<LoginResultDto?> LoginAsync(string email, string password)
{
    var user = await _utilizadorRepo.GetByEmailAsync(email);
    if (user == null || !user.Ativo) return null;
    if (!BCrypt.Net.BCrypt.Verify(password, user.PasswordHash)) return null;

    return new LoginResultDto(user.Id, user.Nome, user.Email, user.Papel, user.LojaId);
}
```

**Regras de negócio:** a verificação de password utiliza `BCrypt.Verify` que é resistente a timing attacks. A mensagem de retorno é idêntica para "utilizador não encontrado" e "password incorrecta", prevenindo enumeração de utilizadores (RF04).

## Serviço de Stock — `StockService`

O `StockService` gere as operações de stock, incluindo a lógica de alerta e a validação de ajustes:

```csharp
public async Task AjustarStockAsync(int lojaId, int produtoId,
    decimal variacao, string motivo, string userId)
{
    var stock = await _stockRepo.GetByLojaAndProdutoAsync(lojaId, produtoId)
        ?? throw new InvalidOperationException("Stock não encontrado");

    var novaQtd = stock.Quantidade + variacao;
    if (novaQtd < 0)
        throw new InvalidOperationException("Stock não pode ficar negativo");

    await _stockRepo.UpdateQuantidadeAsync(stock.Id, novaQtd);
    await _ajusteRepo.CreateAsync(new AjusteStock {
        LojaId = lojaId, ProdutoId = produtoId, UtilizadorId = userId,
        Variacao = variacao, Motivo = motivo, DataHora = DateTime.UtcNow
    });
}
```

## Serviço de Vendas — `SalesService`

O `SalesService` implementa o registo de vendas com validação de stock e atomicidade transaccional:

```csharp
public async Task<VendaDto> CreateSaleAsync(CreateSaleRequest req)
{
    // 1. Validar stock para todos os produtos do carrinho
    foreach (var item in req.Itens)
    {
        var stock = await _stockRepo.GetByLojaAndProdutoAsync(req.LojaId, item.ProdutoId);
        if (stock == null || stock.Quantidade < item.Quantidade)
            throw new InvalidOperationException($"Stock insuficiente: {item.NomeProduto}");
    }

    // 2. Criar cabeçalho de venda
    var venda = await _vendaRepo.CreateAsync(/* ... */);

    // 3. Debitar stock (atómico via transacção EF Core)
    foreach (var item in req.Itens)
        await _stockRepo.AtualizarStockVendaAsync(req.LojaId, item.ProdutoId, -item.Quantidade);

    return venda.ToDto();
}
```

## Background Service — `ConsolidacaoBackgroundService`

O `ConsolidacaoBackgroundService` implementa `BackgroundService` (que por sua vez implementa `IHostedService`), garantindo execução automática sem intervenção humana:

```csharp
protected override async Task ExecuteAsync(CancellationToken stoppingToken)
{
    while (!stoppingToken.IsCancellationRequested)
    {
        var agora = DateTime.Now;
        var alvo = agora.Date.AddHours(23).AddMinutes(59);
        if (agora > alvo) alvo = alvo.AddDays(1);

        await Task.Delay(alvo - agora, stoppingToken);

        try {
            await _consolidacaoService.ConsolidarTodasAsync(DateTime.Today);
        } catch (Exception ex) {
            _logger.LogError(ex, "Falha na consolidação. Retry em 30 min.");
            await Task.Delay(TimeSpan.FromMinutes(30), stoppingToken); // RF34
            await _consolidacaoService.ConsolidarTodasAsync(DateTime.Today);
        }
    }
}
```

## SessionService (Gestão de Sessão RBAC)

O `SessionService` é registado como **singleton** e mantém em memória o estado de sessão de cada utilizador conectado (adequado para Blazor Server, onde cada utilizador tem um circuito SignalR dedicado):

```csharp
public class SessionService
{
    private UserSession? _session;

    public void SetSession(UserSession session) => _session = session;
    public UserSession? GetCurrentUser() => _session;
    public bool IsAuthenticated() => _session != null;
    public bool HasRole(string papel) => _session?.Papel == papel;
    public void Logout() => _session = null;
}
```

O `NavMenu.razor` e todos os componentes Blazor consultam o `SessionService` para mostrar/ocultar elementos conforme o papel do utilizador autenticado.
