# 5.6 Validação do Modelo de Dados

## Processo de Validação

O modelo de dados foi validado de forma iterativa ao longo das fases de design e implementação, utilizando três abordagens complementares:

1. **Revisão cruzada pela equipa** — comparação entre o MER e os RFs para garantir que todos os atributos necessários estão presentes
2. **Prova de conceito com dados reais** — o `SeedData` preencheu o esquema com dados representativos das 5 lojas, confirmando que o modelo suporta todos os fluxos de negócio
3. **Validação através dos testes unitários** — os testes ao `StockService`, `SalesService` e `ProdutoService` exercitam indirectamente a correcta estrutura das tabelas

## Matriz de Cobertura: Requisito → Tabela

| Módulo | RFs Cobertos | Tabelas Necessárias | Cobertura |
|---|---|---|---|
| Auth/RBAC | RF01–RF05 | `Utilizadores` | ✅ |
| Catálogo | RF06–RF10 | `Produtos`, `Categorias` | ✅ |
| Stock | RF11–RF16 | `Stocks`, `AjustesStock` | ✅ |
| POS | RF17–RF22 | `Vendas`, `LinhasVenda`, `Stocks` | ✅ |
| Encomendas | RF23–RF27 | `Fornecedores`, `Encomendas`, `LinhasEncomenda` | ✅ |
| Faturação | RF28–RF32 | `Faturas`, `LinhasFatura` | ✅ |
| Consolidação | RF33–RF36 | `Consolidacoes` | ✅ |
| Relatórios | RF37–RF42 | `Vendas`, `Stocks`, `Consolidacoes` (aggregates) | ✅ |

**Resultado:** Todos os 42 RFs têm suporte no esquema da base de dados. Nenhum RF ficou sem tabela correspondente.

## Verificação através do SeedData

O `SeedData` (em `ConvenienceChain.Data/SeedData.cs`) popula a base de dados com:
- **5 lojas** QuickMart com dados completos
- **3 utilizadores** de diferentes papéis (GestorCadeia, GerenteLoja, Funcionario)
- **5 categorias** hierárquicas
- **20+ produtos** com preços e stock inicial
- **Stock inicial** para cada produto em cada loja com `StockMinimo` definido

A execução da aplicação com o `SeedData` e a navegação por todos os módulos confirmou que o esquema suporta correctamente todos os fluxos operacionais documentados nos Casos de Uso.

## Ajustes Realizados Durante a Implementação

Durante a implementação, foram detectados e corrigidos 2 problemas no modelo inicial:

| Problema | Impacto | Solução |
|---|---|---|
| A tabela `Stocks` não tinha coluna `PrecoVendaLocal` no modelo inicial | RF08 (preço por loja) não era suportado | Adicionada coluna `PrecoVendaLocal DECIMAL(18,2) NULL` |
| O campo `Estado` em `Vendas` não tinha DEFAULT no modelo ER | EF Core gerava erro ao inserir sem especificar estado | Adicionado `DEFAULT 'Concluida'` |

Estes ajustes foram aplicados nas migrações EF Core e reflectidos no dicionário de dados e MER finais.
