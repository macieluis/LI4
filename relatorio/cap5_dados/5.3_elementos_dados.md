# 5.3 Descrição dos Elementos de Dados e Relacionamentos

O Dicionário de Dados documenta em detalhe os atributos de cada tabela, incluindo tipo de dados, restrições e um exemplo representativo. São apresentadas as tabelas mais críticas para a operação do SGCLC.

## Tabela: `Stocks`
> *Entidade central do sistema — regista a quantidade disponível de cada produto em cada loja.*

| Campo | Tipo | Restrições | Descrição | Exemplo |
|---|---|---|---|---|
| `Id` | INT | PK, NN, AUTO_INCREMENT | Identificador único do registo | `101` |
| `LojaId` | INT | FK(Lojas.Id), NN | Loja onde o stock existe | `1` |
| `ProdutoId` | INT | FK(Produtos.Id), NN | Produto ao qual o stock pertence | `42` |
| `Quantidade` | DECIMAL(18,3) | NN, DEFAULT 0 | Quantidade disponível | `150.000` |
| `StockMinimo` | DECIMAL(18,3) | NN, DEFAULT 0 | Limiar de alerta (abaixo → alerta) | `20.000` |
| `PrecoVendaLocal` | DECIMAL(18,2) | NULL | Preço local da loja (override do PrecoBaseVenda) | `0.55` |

> **Restrição de unicidade:** `(LojaId, ProdutoId)` — um produto aparece no máximo uma vez por loja.

## Tabela: `Vendas`
> *Cabeçalho de cada transação de venda registada no POS.*

| Campo | Tipo | Restrições | Descrição | Exemplo |
|---|---|---|---|---|
| `Id` | INT | PK, NN, AUTO_INCREMENT | Identificador único da venda | `5001` |
| `LojaId` | INT | FK(Lojas.Id), NN | Loja onde a venda foi realizada | `2` |
| `FuncionarioId` | NVARCHAR(36) | FK(Utilizadores.Id), NN | Funcionário que registou | `"b2c3d4e5-..."` |
| `DataHora` | DATETIME | NN, DEFAULT GETUTCDATE() | Data e hora da venda | `2026-03-03 14:35:00` |
| `SubTotal` | DECIMAL(18,2) | NN | Soma antes de descontos globais | `12.50` |
| `TotalDesconto` | DECIMAL(18,2) | NN, DEFAULT 0 | Total de descontos aplicados | `1.00` |
| `Total` | DECIMAL(18,2) | NN | Valor final pago | `11.50` |
| `Estado` | NVARCHAR(20) | NN, DEFAULT 'Concluida' | `Concluida`, `Cancelada`, `Devolvida` | `"Concluida"` |
| `MotivoAnulacao` | NVARCHAR(500) | NULL | Motivo (obrigatório se Cancelada/Devolvida) | `"Produto com defeito"` |

## Tabela: `Utilizadores`
> *Contas de utilizador com autenticação e controlo de acesso por papel.*

| Campo | Tipo | Restrições | Descrição | Exemplo |
|---|---|---|---|---|
| `Id` | NVARCHAR(36) | PK | GUID único do utilizador | `"a1b2c3d4-..."` |
| `Nome` | NVARCHAR(200) | NN | Nome completo | `"Ana Rodrigues"` |
| `Email` | NVARCHAR(300) | NN, UK | E-mail de login (único no sistema) | `"ana@quickmart.pt"` |
| `PasswordHash` | NVARCHAR(500) | NN | Hash BCrypt — nunca texto claro | `"$2a$12$..."` |
| `Papel` | NVARCHAR(20) | NN | `GestorCadeia`, `GerenteLoja`, `Funcionario` | `"GerenteLoja"` |
| `LojaId` | INT | FK(Lojas.Id), NULL | Loja associada (null = Gestor da Cadeia) | `1` |
| `Ativo` | BIT | NN, DEFAULT 1 | Conta activa/inactiva | `1` |
| `CriadoEm` | DATETIME | NN, DEFAULT NOW | Timestamp de criação | `2026-02-10 09:00:00` |

## Tabela: `Consolidacoes`
> *Registo dos resultados das consolidações diárias automáticas.*

| Campo | Tipo | Restrições | Descrição | Exemplo |
|---|---|---|---|---|
| `Id` | INT | PK, AUTO_INCREMENT | Identificador único | `501` |
| `LojaId` | INT | FK(Lojas.Id), NN | Loja a que respeita | `2` |
| `DataConsolidacao` | DATE | NN | Dia consolidado | `2026-03-03` |
| `DataHoraExecucao` | DATETIME | NN | Timestamp da execução | `2026-03-03 23:59:05` |
| `TotalVendas` | DECIMAL(18,2) | NN, DEFAULT 0 | Soma das vendas do dia | `1250.75` |
| `NumeroTransacoes` | INT | NN, DEFAULT 0 | Nº de transações | `87` |
| `TotalDescontos` | DECIMAL(18,2) | NN, DEFAULT 0 | Soma de descontos concedidos | `45.20` |
| `Resultado` | NVARCHAR(20) | NN | `Sucesso`, `Falha`, `Parcial` | `"Sucesso"` |
| `ErroDetalhes` | NVARCHAR(2000) | NULL | Mensagem de erro (se aplicável) | `NULL` |

> **Restrição de unicidade:** `(LojaId, DataConsolidacao)` — uma consolidação por loja por dia.

## Convenções Adotadas

| Símbolo | Significado |
|---|---|
| PK | Chave Primária |
| FK | Chave Estrangeira |
| UK | Restrição de Unicidade |
| NN | Not Null (campo obrigatório) |
| NULL | Campo opcional |
| DEFAULT | Valor por omissão |
