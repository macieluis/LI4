# 5.1 Apresentação Geral da Estrutura de Dados

## Abordagem ao Design de Dados

O design da camada de dados do SGCLC seguiu os princípios da modelação relacional, com ênfase na **normalização** (terceira forma normal), na **integridade referencial** enforçada por chaves estrangeiras e nas **restrições de unicidade** que reflectem as regras de negócio do domínio.

O modelo de dados foi desenvolvido em paralelo com a especificação de requisitos (Etapa 1) e refinado durante a fase de design (Etapa 2), resultando num esquema com **11 tabelas principais** que cobrem todos os módulos funcionais do SGCLC.

## Visão Geral do Esquema

```
LOJAS ──────────────────┐
    │                    │
UTILIZADORES            STOCKS ─── PRODUTOS ─── CATEGORIAS
    │                    │
VENDAS ─── LINHASVENDA  AJUSTESSTOCK
    │
FATURAS ─── LINHASFATURA

LOJAS ─── ENCOMENDAS ─── LINHASENCOMENDA
              │
         FORNECEDORES

LOJAS ─── CONSOLIDACOES
```

## Tabelas do Sistema

| Tabela | Descrição | Nº de Atributos | Chaves |
|---|---|---|---|
| `Lojas` | Lojas da cadeia QuickMart | 6 | PK: Id |
| `Utilizadores` | Contas de utilizador | 8 | PK: Id (GUID), UK: Email, FK: LojaId |
| `Categorias` | Categorias e subcategorias de produtos | 3 | PK: Id, FK: CategoriaPaiId (self-ref) |
| `Produtos` | Catálogo de produtos | 9 | PK: Id, UK: Codigo, FK: CategoriaId |
| `Stocks` | Stock por produto por loja | 6 | PK: Id, UK: (LojaId, ProdutoId), FK: LojaId, ProdutoId |
| `Vendas` | Cabeçalho de venda | 8 | PK: Id, FK: LojaId, FuncionarioId |
| `LinhasVenda` | Linhas de produto de uma venda | 6 | PK: Id, FK: VendaId, ProdutoId |
| `Fornecedores` | Fornecedores da cadeia | 7 | PK: Id |
| `Encomendas` | Encomendas a fornecedores | 7 | PK: Id, FK: LojaId, FornecedorId |
| `LinhasEncomenda` | Linhas de produto de uma encomenda | 5 | PK: Id, FK: EncomendaId, ProdutoId |
| `Faturas` | Faturas emitidas | 10 | PK: Id, UK: Numero, FK: LojaId, VendaId |
| `AjustesStock` | Log de auditoria de ajustes manuais | 7 | PK: Id, FK: LojaId, ProdutoId, UtilizadorId |
| `Consolidacoes` | Resultados de consolidações diárias | 9 | PK: Id, UK: (LojaId, DataConsolidacao), FK: LojaId |

## Mapeamento ORM — Entity Framework Core

O Entity Framework Core 8 é utilizado para o mapeamento objeto-relacional. As entidades de domínio em `ConvenienceChain.Core/Models/` mapeiam directamente para as tabelas acima. O `AppDbContext` em `ConvenienceChain.Data/` configura as relações, restrições e conventions através de Fluent API e Data Annotations.

A migração da base de dados em SQLite é aplicada automaticamente no arranque da aplicação (`context.Database.EnsureCreated()` + `SeedData.Initialize()`), garantindo que o ambiente de desenvolvimento fica pronto sem configuração manual.
