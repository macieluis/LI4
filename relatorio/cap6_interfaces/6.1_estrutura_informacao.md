# 6.1 Estrutura Geral e Arquitetura de Informação

## Filosofia de Design de Interfaces

O design de interfaces do SGCLC seguiu dois princípios orientadores:

1. **"Role-first" navigation** — o menu de navegação e as funcionalidades disponíveis são determinados pelo papel do utilizador autenticado. Um Funcionário de POS nunca vê o menu de Relatórios; um Gestor da Cadeia nunca vê o botão de Confirmar Venda.

2. **"Density by role"** — a complexidade da interface escala com o papel: o Funcionário tem um POS simples e focado; o Gerente tem dashboards com KPIs e múltiplos módulos; o Gestor tem a visão mais densa com dados de todas as lojas.

## Arquitectura de Informação — Mapa de Navegação

```
SGCLC
├── Login (todos)
│
├── [Funcionário]
│   ├── Dashboard (básico)
│   └── POS
│       ├── Tabela produtos disponíveis (stock > 0)
│       ├── Filtro instantâneo
│       ├── Carrinho + desconto
│       └── Confirmação + recibo
│
├── [Gerente de Loja]
│   ├── Dashboard (KPIs da loja)
│   ├── POS
│   ├── Stock
│   │   ├── Listagem com alertas
│   │   └── Ajuste manual
│   ├── Encomendas
│   │   ├── Lista + estados
│   │   ├── Nova encomenda
│   │   └── Receção
│   ├── Faturas
│   └── Relatórios (loja)
│
└── [Gestor da Cadeia]
    ├── Dashboard (global – todas as lojas)
    ├── Stock (todas as lojas)
    ├── Encomendas (todas as lojas)
    ├── Faturas (todas as lojas)
    ├── Relatórios (global + comparativo)
    ├── Consolidação
    │   ├── Histórico de consolidações
    │   └── Trigger manual
    └── Administração
        └── Utilizadores (CRUD)
```

## Layout Global da Aplicação

Todas as páginas partilham o mesmo layout base (`MainLayout.razor`) com três zonas:

```
┌─────────────────────────────────────────────────────────────────┐
│  🏪 SGCLC – QuickMart    [Loja: Braga-Centro]  👤 Ana R.  [↩] │  ← NavBar
├────────────────┬────────────────────────────────────────────────┤
│                │                                                 │
│  📊 Dashboard  │                                                 │
│  🛒 POS        │          ÁREA DE CONTEÚDO PRINCIPAL            │
│  📦 Stock      │         (varia conforme a página)              │
│  📋 Encomendas │                                                 │
│  🧾 Faturas    │                                                 │
│  📈 Relatórios │                                                 │
│                │                                                 │
└────────────────┴────────────────────────────────────────────────┘
```

O **NavBar** mostra sempre: nome da aplicação, loja activa (para Gerentes e Funcionários) e botão de logout. O **sidebar** adapta os seus itens ao papel do utilizador conforme a matriz RBAC documentada na secção 2.4. A **área de conteúdo** ocupa o espaço restante e é renderizada pelo router Blazor.

## Princípios de Consistência Visual

- **Alertas de stock**: sempre em laranja/amarelo 🟡 (alerta) e verde ✅ (OK), consistentemente em todas as páginas que mostram stock
- **Estados de entidades**: cores consistentes — verde (Concluída/Rececionada), laranja (Pendente/Enviada), vermelho (Cancelada)
- **Botões de acção primária**: sempre azul (Bootstrap `btn-primary`)
- **Botões de perigo/cancelar**: sempre vermelho (Bootstrap `btn-danger`)
- **Formulários**: todos com validação client-side (Bootstrap validation classes)
