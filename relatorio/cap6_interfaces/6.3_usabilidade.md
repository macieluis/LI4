# 6.3 Usabilidade e Acessibilidade

## Princípios de Usabilidade Aplicados

O design de interfaces do SGCLC seguiu os **10 heurísticos de usabilidade de Nielsen** como referência. São descritos os mais relevantes para o contexto do sistema:

### Visibilidade do Estado do Sistema

O utilizador está sempre informado sobre o que está a acontecer:
- O carrinho do POS mostra em tempo real o número de itens e o total
- As badges de stock (🟡 alerta / ✅ OK) são visíveis em todas as listagens de produtos
- Os estados das encomendas e faturas estão sempre visíveis com cores distintas
- O resultado da consolidação (Sucesso / Falha) é imediatamente visível após a execução

### Correspondência com o Mundo Real

A terminologia utilizada na interface foi validada com os stakeholders:
- "POS" em vez de "terminal de vendas" — linguagem familiar aos funcionários
- "Stock mínimo" em vez de "nível de reordenamento" — termo usado pelo cliente
- "Grosso de encomenda" evitado — usam-se "encomenda" e "receção"

### Controlo e Liberdade do Utilizador

- O botão **"Limpar Carrinho"** no POS permite ao Funcionário cancelar uma venda antes de confirmar
- As vendas concluídas podem ser canceladas pelo Gerente (com motivo obrigatório)
- Os formulários têm sempre botão "Cancelar" para sair sem guardar

### Prevenção de Erros

- A confirmação de venda valida o stock de **todos** os produtos antes de processar
- O campo "Motivo" é obrigatório em ajustes de stock e cancelamentos, prevenindo operações não documentadas
- O sistema impede quantidades negativas de stock (validação ao nível do serviço e da base de dados)

### Eficiência de Uso (Utilizadores Experientes)

- O filtro do POS é activado com o primeiro caractere digitado — sem botão "Pesquisar"
- A adição ao carrinho é feita com um único clique no botão "+"
- O Dashboard mostra os KPIs mais importantes sem necessidade de navegação adicional

## Acessibilidade

| Critério | Implementação |
|---|---|
| **Cores não como única forma de informação** | As badges de stock usam ícone (🟡/✅) + cor + texto da quantidade |
| **Contraste adequado** | Bootstrap 5 garante contraste mínimo WCAG 2.1 AA |
| **Responsividade** | Layout Bootstrap 5 responsivo, utilizável em ecrãs ≥ 1024px |
| **Fontes legíveis** | Fonte padrão Bootstrap (system-ui) com tamanho base 16px |
| **Inputs com labels** | Todos os campos de formulário têm `<label>` associado |

## Avaliação de Usabilidade

Foram realizados testes informais de usabilidade com dois utilizadores não treinados:

| Tarefa | Utilizador | Resultado | Tempo |
|---|---|---|---|
| Registar uma venda de 3 produtos no POS | Utilizador A (não técnico) | ✅ Concluído sem ajuda | 45 segundos |
| Criar uma encomenda a um fornecedor | Utilizador B (técnico) | ✅ Concluído | 2 minutos |
| Consultar os alertas de stock no Dashboard | Utilizador A | ✅ Encontrou imediatamente | 5 segundos |

**Conclusão:** O POS cumpre o RNF07 (operável em menos de 60 segundos por utilizador não treinado). A interface de encomendas é mais complexa mas dentro do esperado para o perfil de Gerente de Loja.
