## Stack Tecnológica

| Camada | Tecnologia | Justificação |
|---|---|---|
| **Framework Web** | ASP.NET Core 8.0 – Blazor Server | Renderização server-side com SignalR, sem necessidade de API REST separada |
| **ORM** | Entity Framework Core 8.0 | Mapeamento objeto-relacional tipado, suporte a SQLite e SQL Server |
| **Base de Dados (dev)** | SQLite | Zero-configuration, portável, suficiente para desenvolvimento e demonstração |
| **Base de Dados (prod)** | SQL Server 2019+ | Escalabilidade, ACID, suporte a múltiplas ligações concorrentes |
| **Linguagem** | C# 12 | Tipagem estática, suporte nativo a async/await, ecosistema .NET |
| **UI / CSS** | Bootstrap 5.3 + Bootstrap Icons | Responsivo, componentes prontos, consistência visual |
| **Hashing de Passwords** | BCrypt.Net-Next | Função de hash segura com salt automático (custo configurável) |
| **Testes** | xUnit + Moq + FluentAssertions | Framework de testes mais utilizado em .NET, mocking limpo |
| **Controlo de Versões** | Git + GitHub | Gestão de código, histórico de commits, colaboração em equipa |

## Estrutura do Projecto .NET

O projecto está organizado em três assemblies .NET com separação clara de responsabilidades (Clean Architecture/Ports & Adapters):

```
src/
├── ConvenienceChain.Core/       ← Entidades de domínio, Interfaces, DTOs, Serviços
├── ConvenienceChain.Data/       ← DbContext EF Core, Repositórios, Seed Data
└── ConvenienceChain.Web/        ← Blazor Components, SessionService, Program.cs
```

| Assembly | Responsabilidade | Dependências |
|---|---|---|
| `ConvenienceChain.Core` | Domínio, contratos de serviço (interfaces), DTOs | Nenhuma dependência de infraestrutura |
| `ConvenienceChain.Data` | Acesso à base de dados, contexto EF Core, repositórios | Depende de Core |
| `ConvenienceChain.Web` | Interface Blazor, injeção de dependências, configuração | Depende de Core e Data |
